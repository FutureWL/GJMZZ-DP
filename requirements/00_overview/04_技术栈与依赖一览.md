## 2. 各 App 依赖差异(避免重复)

| App | 关键额外依赖 |
|---|---|
| `portal-ui`(PC 门户) | echarts、echarts-for-react、recharts、react-hook-form、zod、@hookform/resolvers |
| `mobile-portal-ui`(移动 H5) | keycloak-js(无 recharts/echarts,适合低端机) |
| `cockpit-ui`(驾驶舱大屏) | recharts(图表大屏,无表单库) |
| `mobile-app`(原生) | @capacitor/{core,cli,android,browser}、@tanstack/react-query、zustand、date-fns |
| `factory-api`(后端) | @nestjs/{common,core,platform-fastify,swagger}、@prisma/client、@fastify/static、jose(JWT 校验)、pg、fastify、dotenv |

## 3. 共享包

- `packages/mock-data`(`@factory/mock-data`):
  - `models.ts` 全局 TS 模型(Alarm、WorkOrder、TraceRecord、PurchaseRequest、Supplier、ItTicket、Customer、Opportunity、MesDispatch、QualityTask、Material、InventoryItem、Expense、Department、Project、CostCenter、ExpenseClaim、ProcurementRequest、ContractReview、ApprovalWorkItem、NotificationItem、AuditLogEntry、Contact、CrmActivity、Quote、QuoteLine、AdditionalCenter/Service/Request/Content/Role、Factory、MorningMeetingKpi、MorningRisk、DeliveryRisk、Incident、EquipmentAsset、MaintenanceTicket 等)
  - `data.ts` / `additional.ts` / `orders.ts` 业务样例数据
  - 由 `portal-ui` / `mobile-portal-ui` / `cockpit-ui` workspace 引入

## 4. 关键版本注意

- 仓库里 `typescript: ~6.0.2`、`vite: ^8.0.12`、`eslint: ^10.3.0` 看起来是**未来版本号**(TS 截至当前主线 5.x、Vite 6/7、ESLint 9),疑似本地环境使用的是自构建/fork 版本。开发时若遇版本解析问题,优先复用现有 lockfile(`pnpm-lock.yaml`)。
- React 19.2.x 同样是较新主线,需关注 `react-router-dom@7` 的兼容。
- `keycloak-js@26.0.5` 与 Keycloak server 26.0.5 对齐,SDK 与 server 必须大版本一致。
