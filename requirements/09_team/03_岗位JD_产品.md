# 03 岗位 JD · 产品组

> 本文档定义 **产品组** 的岗位 JD(产品经理、UI/UX 设计师)。
> 配套:[01 总览](./01_团队组建总览.md) · [02 岗位体系](./02_组织架构与岗位体系.md) · [04-06 其他组 JD](./04_岗位JD_技术.md) · [07 薪酬体系](./07_薪酬体系.md)

## 通用要求(产品组)

- **行业经验**:B2B 复杂信息系统(ERP/MES/QMS/WMS/SRM 任一)产品经验**优先**;
- **学历**:本科及以上;
- **沟通**:能听懂"工厂语言"(工艺/排产/质量/设备),并翻译成 PRD;
- **文档**:PRD 写作清晰、结构化、可执行;
- **抗压**:能接受"开 5 个会才推 1 件事"的节奏(制造业 IT 决策链长);
- **价值观**:用户视角 > 老板视角(尤其是"业务部门领导说什么就做什么"陷阱)。

---

## 🔴 P0-01:产品经理(B2B 复杂系统)

**【岗位职责】**

1. **产品规划**:
   - 负责 GJMZZ-DP 平台 5 大域(经营/管理/生产/支持/附加)+ 3 层(高/中/基层)的产品规划;
   - 调研竞品(用友 U9/金蝶云·星瀚/鼎捷 ERP/宝信 MES/中控 i-OMC/SAP ME 等),输出竞品分析报告;
   - 与制造业务架构师协作,把工厂真实需求转化为产品形态;
   - 制定产品路线图,与 CEO/客户对齐。

2. **需求管理**:
   - 收集用户(15 个业务角色:总经理/制造副总/班组长/检验员/维修技师等)需求,建立需求池;
   - 编写 PRD(含功能、流程图、原型说明、验收标准);
   - 组织需求评审,与技术、设计、制造业务架构师达成共识;
   - 维护需求优先级,推动 MVP/Phase 1-3 功能交付。

3. **项目推动**:
   - 主导产品迭代节奏(2 周 Sprint / 月度版本);
   - 跨部门协调(技术、设计、教研[本项目不做]、运营、实施);
   - 跟进开发进度,及时解决阻塞问题。

4. **数据分析**:
   - 定义产品核心指标(用户活跃度、模块使用率、流程时效、SLA 等);
   - 搭建数据看板,每周/月输出数据报告;
   - 基于数据持续优化产品。

5. **用户研究**:
   - 定期组织用户访谈(总经理/制造副总/班组长/一线操作员);
   - 设计调研问卷,输出用户研究报告;
   - 跟进 NPS(净推荐值)等满意度指标。

**【任职要求】**

- **学历**:本科及以上;
- **经验**:3 年以上 B2B 复杂信息系统产品经理经验;
- **行业**:有 ERP/MES/QMS/WMS/SRM/PLM/HR 系统中**至少一个**的完整实施经验(不只是"调研过");
- **能力**:
  - 优秀的需求分析、产品设计能力(能画清楚流程图、状态机);
  - 熟练使用 Figma / Axure / 墨刀 / ProcessOn;
  - 良好的文档能力(PRD 写作清晰、结构化);
  - 良好的跨部门沟通协调能力(尤其要懂"工厂的车间主任说话方式");
  - 数据敏感,能用数据驱动决策;
- **素质**:
  - 用户视角(特别能理解"一线班组长"和"总经理"两个极端);
  - 抗压能力(制造业决策慢,要会等);
  - 不怕"脏活"(B2B 经常要陪客户蹲车间)。

**【加分项】**

- 有制造业 IT 产品经验(用友/金蝶/鼎捷/宝信/中控出身);
- 有从 0 到 1 的产品经验;
- 有 BPM/工作流产品经验(加分);
- 懂基础 SQL(能自己跑数据)。

**【薪酬范围】** ¥15K-25K/月(13 薪) + 期权(视情况)

## 🔴 P0-02:UI/UX 设计师

**【岗位职责】**

1. **界面设计**:
   - 负责 4 个前端应用(portal-ui / mobile-portal-ui / cockpit-ui / mobile-app)的 UI 设计;
   - 基于设计规范(`design/System/01_DesignTokens_色彩字体间距.md`),统一视觉风格;
   - 输出高保真原型(Figma),交付给前端开发;
   - 重点关注:**大屏可视化**(cockpit-ui)、**移动端效率**(mobile-app)、**复杂表单**(portal-ui)。

2. **设计系统**:
   - 搭建并维护设计系统(设计 Tokens + 组件库),与 `design/System/` 同步;
   - 持续优化设计规范,提升团队效率;
   - 关注 B2B 复杂信息系统设计趋势,引入新设计理念(如"看板密度优化""高密度信息布局")。

3. **交互设计**:
   - 关键交互流程的动效设计;
   - 用户体验优化(提升信息密度、降低操作步骤);
   - 配合产品经理输出交互原型。

4. **品牌物料**(辅助):
   - 营销物料(客户案例、产品手册、官网 banner);
   - 投标用 PPT 模板(制造业销售重要);
   - 公司 VI 物料基础设计。

**【任职要求】**

- **学历**:本科及以上(设计相关专业);
- **经验**:3 年以上 B2B 复杂信息系统 UI 设计经验;
- **能力**:
  - 精通 Figma(必须)、Sketch、PS、AI;
  - 熟悉 B2B 设计语言("高密度信息""数据优先""专业感");
  - 有大屏可视化设计经验(加分);
  - 有移动端 App 设计经验(加分);
  - 良好的审美和细节把控能力;
- **作品**:提供高质量作品集(必须有 B2B 案例,纯 C 端作品不收)。

**【加分项】**

- 有 ERP/MES/QMS 类系统设计经验;
- 有数据可视化(cockpit / 大屏)经验;
- 有动效设计能力(After Effects / Lottie);
- 有设计系统搭建经验(加分)。

**【薪酬范围】** ¥12K-20K/月(13 薪)

## 🟡 P1-01:数据分析师(Phase 1-2 招)

**【岗位职责】**

1. **埋点分析**:
   - 配合产品经理定义北极星指标、漏斗、留存;
   - 搭建埋点规范,审核前端埋点代码;
   - 输出用户行为分析报告。

2. **A/B 测试**:
   - 设计与分析 A/B 测试(如新功能上线对比);
   - 输出显著性结论。

3. **业务分析**:
   - 用户画像 / RFM 分层(按"角色 × 工厂 × 部门");
   - 周报 / 月报 / 业务分析报告;
   - 与产品/实施协作,验证功能效果。

**【任职要求】**

- 3-5 年数据分析经验;
- 熟练 SQL + Python + 至少一种 BI 工具(Tableau / PowerBI / Metabase);
- 有用户增长分析经验优先;
- **懂制造业 KPI 体系**(产能/良率/OEE/MTBF/MTTR 等)优先。

**【薪酬范围】** ¥12K-22K/月(13 薪)

## 附录:产品组的"反模式"

> 以下情况在面试时要警惕:

| 反模式 | 表现 | 提问方式 |
|---|---|---|
| PPT 产品经理 | 只会画原型,不懂实现 | "如果让你做一个工单状态机,你怎么跟后端描述?" |
| C 端思维 | 总想做"用户增长""拉新" | "B 端软件的留存和 C 端有什么区别?" |
| 老板依赖 | 老板说什么就做什么 | "如果总经理和班组长需求冲突,你怎么处理?" |
| 不会写 PRD | 只会画图 | "你写过最长的 PRD 是多少页?结构是什么样?" |
| 怕工厂 | 不愿意下车间 | "你之前去过工厂吗?车间什么样?" |
