# 01 团队组建总览

## 1. 一句话目标

> 在 **12 个月内**,为 GJMZZ-DP 平台组建一支 **14-22 人** 的"工业软件精兵",
> 完成从"5 套 App + 1 套 BPMN + 15 个种子用户"到"5 大业务域全打通 + 真实工厂试点落地"的跨越。

## 3. 时间表与阶段重点

> 与项目里程碑对齐(参考项目根 `CHANGELOG.md` 与 GitHub Issue 的 Backlog)。

| 阶段 | 时间 | 团队规模 | 重点 |
|---|---|---|---|
| **MVP** | M0-M3(0-3 月) | **14-16 人** | 补齐核心技术 + 制造业务首批 PM/BA + DevOps + 实施 |
| **Phase 1** | M3-M6(3-6 月) | **16-19 人** | +移动端深化 + 实施工程师 + 客户成功 |
| **Phase 2** | M6-M9(6-9 月) | **18-22 人** | +OT/SCADA 集成 + 大屏可视化 + DBA + 安全 |
| **Phase 3** | M9-M12(9-12 月) | **20-26 人** | +BI/数据 + 第二个客户成功 + 法务/合规 |

> **注意**:本方案不是"招满 22 人"为目标,而是**按项目里程碑刚性需求招人**。每个 Phase 结束 review 一次。

## 4. 与参考模板的差异(为什么不能照搬)

> 本项目不是"少儿编程教育平台",故不能直接套用 `zhiye-code/docs/08-团队组建方案.md`。
> 主要差异:

| 维度 | 参考模板(少儿编程) | **GJMZZ-DP(高精密制造)** | 影响 |
|---|---|---|---|
| 业务形态 | B2C/B2B2C,6 校区线下 | B2B 企业级,工厂客户 | 团队规模可压缩 50% |
| 用户 | 家长+小孩 | 15 种业务角色(3 层 5 域) | 需懂角色化设计 |
| 核心能力 | 关卡编辑器、Scratch Blocks、AI 学伴 | BPM(Flowable)、OT/SCADA、TDengine 时序、大屏 | 技术栈完全不同 |
| 销售/交付 | 校区校长+教务 | 实施工程师+客户成功 | 售前/售后不同 |
| 教研/内容 | 关卡设计师、视频制作 | **不需要**(本项目不做教育内容) | 移除相关 JD |
| 教学/学习 | 教研负责人、教研组长 | **不需要** | 移除相关 JD |
| 工业合规 | 不涉及 | 等保 2.0、数据安全法、ISO 9001/IATF 16949 | 需安全/合规岗 |
| 城市/薪酬 | 南昌(60-70% 一线) | **苏州/上海/深圳**(70-85% 一线) | 薪酬级别表重定 |
| 远程 | 教研必须现场 | 销售/实施/客户成功 **必须现场**(去工厂) | 远程策略不同 |
| 股权/期权 | 15-20% 池 | 10-15% 池(创始人 + 1-2 个联创) | 创业期精简 |

## 5. 招聘原则(五大原则)

### 5.1 一专多能

- **核心岗位**(技术负责人、BPM 平台工程师、制造业务架构师)必须有**专精深度**(5 年+ 单一栈);
- **配套岗位**(前端、后端、测试)可以**跨域协作**(全栈 / BFF 思维);
- **业务岗**(PM、BA、实施)必须**懂工厂**(有制造业 IT 项目落地经验)。

### 5.2 先精后全

- 先招 **核心技术 + 业务架构** 这 4-5 个人,**不惜代价**(包括猎头/外包);
- 再补 **执行层**(前端、后端、测试、实施);
- 最后补 **运营/支持层**(行政、HR、客服)。

### 5.3 文化优先

- 价值观一致 > 技术能力;
- 制造业 IT 是**慢活**,招愿意"沉下心"做 3 年的人,而不是"3 个月一跳"的;
- 警惕**"PPT 架构师"**:能讲不能做,来了就变成瓶颈。

### 5.4 实战经验

- 优先有**完整制造业 IT 项目落地经验**(MES / ERP / WMS / SRM / QMS 任一)的人;
- **不强求大厂背景**——但要求有**真工厂交付案例**(能说出 1-2 个具体项目);
- **应届生只招"实习转正"**,不直接社招。

### 5.5 成长潜力

- 看**学习能力 + 业务理解** > 看当前水平;
- 制造业 IT 技术更新慢,但**业务复杂**(工艺/排产/质量/设备联动),所以招能"啃硬骨头"的人;
- 警惕**纯互联网背景**(可能不耐受"开 5 个会才推 1 件事"的节奏)。

## 6. 远程 vs 现场

| 岗位 | 现场 | 远程 |
|---|---|---|
| CEO / 联创 | ✅ 必须 | — |
| 技术负责人 / 架构师 | ✅ 必须 | — |
| 业务架构师(制造) | ✅ 必须(去工厂) | — |
| 前端 / 后端 / DevOps | ⚠️ 优先现场 | ✅ Phase 1 后可灵活 |
| 产品经理(中后台) | ⚠️ 优先现场 | ✅ 可灵活 |
| UI 设计师 | ⚠️ 优先现场 | ✅ 可灵活 |
| QA / DBA | ⚠️ 优先现场 | ✅ Phase 1 后可灵活 |
| 实施工程师 | ✅ **必须现场**(去客户工厂) | — |
| 客户成功 | ✅ **必须现场**(驻场/出差) | — |
| HR / 行政 | ✅ 现场 | — |
| 法务 / 财务(兼职) | — | ✅ 可远程 |

> **建议**:MVP 阶段**核心技术团队 + 实施必须现场**;Phase 1 后**视项目成熟度**灵活安排。

## 7. 招聘策略

### 7.1 MVP 期策略(M0-M3)

- **核心研发靠社招**(技术负责人、Flowable BPM 工程师、后端核心、前端核心);
- **制造业务靠"挖角"**——从制造业 IT 厂商(用友/金蝶/鼎捷/宝信/中控)挖有 3-5 年项目经验的人;
- **设计/产品靠"借脑"**——初期可考虑兼职/外包 1-2 个 B2B 资深产品/设计师;
- **慎用外包**——核心岗位不用,外包只做**单点可交付物**(如 BPMN 流程图细化、Prisma 迁移脚本)。

### 7.2 Phase 1+ 策略

- **关键岗位继续社招**;
- **批量岗位可考虑校招**(前端/后端/测试,要求是 211/985 计算机相关专业);
- **实施/客户成功可从制造业 IT 乙方转**(用友/金蝶的实施顾问最匹配);
- **关键岗位必要时使用猎头**(技术负责人、Flowable BPM 工程师、制造业业务架构师)。

## 8. 不做清单(避免误招)

| 不招的岗位 | 原因 |
|---|---|
| 关卡设计师 / 教学设计师 / 信奥教练 | 本项目不做教育内容 |
| 视频制作 / 动画师 / 配音师 | 本项目不做课程视频 |
| 内容老师(兼职) | 本项目不做教研 |
| 校区运营总监 / 校区校长 | 本项目不做线下校区 |
| 社群运营 / 新媒体运营 | 本项目不做 C 端拉新 |
| 市场总监 / BD 销售经理 | 初期由 CEO/联创亲自做销售,不做大规模 BD |
| AI 算法工程师 | **本期不投入 AI**(理由:制造业 AI 是 Phase 3 之后的事,先做"数字化的底座") |

## 9. 关键约束(必须在团队组建时同步做)

1. **法务合规**——制造业涉及等保 2.0 / 数据安全法 / 招投标法 / 合同法,**法务顾问必须 M1 内到位**(可以外包/兼职);
2. **数据安全**——制造业客户对数据本地化敏感,**所有架构决策必须考虑"私有化部署"**;
3. **实施能力**——B2B 软件必须**有人去客户工厂**,不能纯远程,实施工程师 M1 开始就要招;
4. **7×12 工单支持**——生产域告警 7×12(早 8:00-晚 22:00)响应,**M3 必须有值班机制**;
5. **多客户/多租户**——虽然本期单租户,**架构上要为多租户留口子**(数据库 schema、SSO realm、文件存储)。
