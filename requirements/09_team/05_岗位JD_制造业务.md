# 05 岗位 JD · 业务组(制造业务)

> 本文档定义 **业务组(制造业务架构师 + 域 BA)** 的岗位 JD。
> 这是 GJMZZ-DP 项目**最特色**的岗位——**把工厂语言翻译成产品需求的人**。
> 配套:[01 总览](./01_团队组建总览.md) · [02 岗位体系](./02_组织架构与岗位体系.md) · [06 实施与支持 JD](./06_岗位JD_运营与支持.md)

## 为什么单独有"业务组"

> B2B 工业软件最大的难题是"业务理解"——
> - 产品经理不懂工艺,写不出"工序""BOM 展开""不良率"的 PRD;
> - 工程师不懂排产规则,做不出"瓶颈识别"算法;
> - 实施不懂客户,落地就鸡同鸭讲。
>
> 因此单独设"业务组"——招**真正在制造业 IT 干过 5+ 年**的人,做"产品 ↔ 客户"的桥梁。

## 通用要求(业务组)

- **行业经验**:**必须**有制造业 IT 完整项目经验(用友/金蝶/鼎捷/宝信/中控/蓝凌等出身,或甲方 IT 部门出身);
- **业务知识**:能讲清楚"工单/排产/SPC/OEE/MTBF/MTTR"等术语;
- **沟通**:能跟"车间主任"和"IT 总监"都用对方听得懂的话沟通;
- **抗压**:愿意"陪客户蹲车间"(经常要出差);
- **学习**:愿意学新技术(虽然不是写代码,但要懂 BPMN / 流程引擎 / 简单 SQL);
- **价值观**:"懂客户" > "懂技术"——警惕"卖软件思维"。

## 🔴 P0-01:制造业务架构师(**关键岗位**)

> **为什么是关键岗位**:B2B 工业软件初期,客户判断"你们懂不懂行"就看这个人。
> 这个人必须能跟制造副总、生产经理"对上话"——讲工艺、讲排产、讲质量。

**【岗位职责】**

1. **业务架构**:
   - 主导 GJMZZ-DP 5 大域(经营/管理/生产/支持/附加)的业务架构设计;
   - 抽象工厂的"业务对象"(工单/批次/序列/设备/检验/告警/事故/外协等);
   - 设计"业务流程"(生产流转、质量闭环、设备维修、采购流程、审批流程);
   - 输出业务架构文档(业务流程图、状态机、规则清单)。

2. **领域建模**:
   - 主导 Prisma 数据模型设计(与后端核心协作);
   - 抽象"通用业务能力"(组织/工厂/产线/部门主数据,通用字典);
   - 设计"业务规则"(SPC 规则、排产规则、告警规则)。

3. **客户对接**:
   - 作为"首席业务顾问"对接客户高层(总经理/制造副总);
   - 主导需求调研(用户访谈、流程梳理、KPI 拆解);
   - 把客户需求转化为产品需求(与产品经理协作)。

4. **售前支持**:
   - 配合 CEO/销售做客户拜访;
   - 主导业务方案撰写(投标书、解决方案、白皮书);
   - 客户高层培训(总经理闭门会、制造副总 workshop)。

5. **团队赋能**:
   - 培训域 BA(生产/质量/设备);
   - 培训产品经理的制造业知识;
   - 培训实施工程师的客户沟通技巧。

**【任职要求】**

- **学历**:本科及以上(工业工程/管理/计算机/自动化优先);
- **经验**:8 年以上制造业 IT 经验,**至少 3 个完整项目**(从 0 到 1);
- **行业**:有 MES / QMS / WMS / SRM / ERP 任一系统的实施或产品经验;
- **能力**:
  - **深厚的制造业业务知识**(能讲清楚离散制造 vs 流程制造 vs 单元制造的差异);
  - 优秀的需求分析、流程梳理能力(能用 BPMN 画清楚流程);
  - 良好的客户沟通能力(尤其面对客户高层);
  - 良好的文档能力(业务方案、解决方案、白皮书);
  - 懂基础数据建模(能看懂 ER 图);
- **行业经验**:
  - 离散制造(机加工/电子/汽车零部件/装备制造)优先;
  - 熟悉 ISO 9001 / IATF 16949 / ISO 14001 体系优先;
  - 熟悉精益生产 / 六西格玛 / 5S 优先;
- **加分**:
  - 有甲方工厂 IT 经验(在工厂里干过 3+ 年);
  - 有乙方实施顾问经验(用友/金蝶/鼎捷/宝信出身);
  - 有 PMP / 信息系统项目管理师认证;
  - 懂 BPMN / DMN 建模。

**【薪酬范围】** ¥25K-40K/月(13 薪)+ 期权(关键岗位)

## 🟡 P1-01:生产域 BA(Business Analyst)

**【岗位职责】**

1. **生产域需求**:
   - 负责生产域(工厂总览/工单/排程/质量/设备/告警/追溯)的需求分析;
   - 与制造业务架构师协作,深入车间调研;
   - 编写详细需求规格(SRS / User Story / BPMN)。

2. **业务规则**:
   - 抽象生产域的业务规则(排产规则、SPC 规则、告警规则、状态机);
   - 与 Flowable BPM 工程师协作,转化为 BPMN;
   - 维护"业务规则清单"(核心交付物)。

3. **UAT 验收**:
   - 主导生产域功能的 UAT 验收(参考 `08_role-journeys/04_acceptance-checklist-middleground.md`);
   - 陪客户蹲车间做测试;
   - 输出验收报告。

4. **培训**:
   - 培训车间主任/班组长使用新功能;
   - 收集一线反馈,持续优化。

**【任职要求】**

- **学历**:本科及以上(工业工程/管理/计算机/自动化优先);
- **经验**:5 年以上制造业 IT 经验(MES / ERP 任一);
- **能力**:
  - 熟悉生产管理(工单/排程/报工/工序);
  - 熟悉质量管理(SPC / 不良分析 / 追溯);
  - 熟悉设备管理(OEE / 维保 / 备件);
  - 良好的客户沟通能力(尤其面对车间主任/班组长);
  - 良好的文档能力;
- **行业经验**:离散制造优先;
- **加分**:
  - 懂精益生产 / 5S;
  - 懂 BPMN;
  - 有 ISO 9001 内审员证。

**【薪酬范围】** ¥12K-20K/月(13 薪)

## 🟡 P1-02:质量域 BA

**【岗位职责】**

1. **质量域需求**:
   - 负责质量域(检验任务/SPC/不良分析/追溯)的需求分析;
   - 深入质量部 / 实验室 / 车间检验岗调研;
   - 编写详细需求规格。

2. **质量业务规则**:
   - 抽象质量管理规则(抽样规则、SPC 控制图规则、不良判定规则、追溯链);
   - 与 Flowable BPM 工程师协作,把"质量异常处理"转化为 BPMN;
   - 维护"质量规则库"。

3. **QMS 集成**:
   - 调研客户现有 QMS(如有用友/金蝶/SAP);
   - 设计数据接口(检验数据/不良数据/追溯数据);
   - 评估与 ISO 9001 / IATF 16949 / VDA 6.3 的对齐。

4. **UAT 验收 + 培训**:
   - 同生产域 BA。

**【任职要求】**

- 5 年以上制造业 IT 经验(QMS 优先);
- 熟悉质量管理(SPC / FMEA / PPAP / APQP / 8D);
- 熟悉 ISO 9001 / IATF 16949 体系;
- 良好的客户沟通能力(尤其面对质量经理/检验员);
- **加分**:
  - 有 CQE / CQA / 六西格玛黑带认证;
  - 有汽车行业 IATF 16949 经验;
  - 懂统计学(基础 SPC 规则)。

**【薪酬范围】** ¥12K-20K/月(13 薪)

## 🟢 P2-01:设备域 BA(Phase 2 招)

**【岗位职责】**

1. **设备域需求**:
   - 负责设备域(设备台账/设备监控/OEE/维保/维修工单)的需求分析;
   - 深入设备部 / 维修车间调研;
   - 编写详细需求规格。

2. **OT/SCADA 集成**:
   - 调研客户现有 SCADA / DCS / PLC 系统;
   - 设计数据接口(OPC UA / Modbus / MQTT);
   - 评估 OEE 计算规则;
   - 与 DevOps / 后端核心协作,实现"设备 → 平台"的实时数据流。

3. **设备业务规则**:
   - 抽象设备管理规则(维保计划规则、备件库存规则、维修优先级);
   - 维护"设备规则库"。

4. **UAT 验收 + 培训**:
   - 同生产域 BA。

**【任职要求】**

- 5 年以上制造业 IT 经验(设备管理 / EAM / CMMS 优先);
- 熟悉设备管理(OEE / TPM / MTBF / MTTR);
- 熟悉工业协议(OPC UA / Modbus / MQTT);
- 良好的客户沟通能力(尤其面对设备经理/维修技师);
- **加分**:
  - 有 OT/SCADA 集成经验;
  - 懂 PLC(西门子/三菱/欧姆龙任一);
  - 懂工业网络(Profinet / EtherCAT);
  - 懂设备预测性维护(AI 方向,Phase 3 备用)。

**【薪酬范围】** ¥12K-20K/月(13 薪)

## 🟢 P2-02:管理域 / 经营域 BA(Phase 2 招,可选)

**【岗位职责】**

1. **管理域 + 经营域需求**:
   - 负责管理域(审批中心/采购/供应商/外协)和经营域(驾驶舱/分析)的需求分析;

2. **BPM 流程**:

   - 维护"管理流程库"。

3. **报表**:
   - 设计经营分析报表(KPI 体系);
   - 配合数据分析师输出可视化。

**【任职要求】**

- 5 年以上制造业 IT 经验(ERP / SRM / 财务系统优先);
- 熟悉制造业经营管理(KPI / 预算 / 成本);
- 熟悉 BPM 工作流;
- **加分**:有财务/采购业务背景。

**【薪酬范围】** ¥12K-20K/月(13 薪)

## 🟢 P3-01:工业数据分析师(Phase 3 招)

**【岗位职责】**

1. **生产数据分析**:
   - 产能分析(OEE / 节拍 / 瓶颈);
   - 质量分析(SPC / 不良 / 缺陷 TopN);
   - 设备分析(MTBF / MTTR / 停机原因);
   - 经营分析(回款 / 毛利 / 客户)。

2. **TDengine 高级应用**:
   - 设计时序数据模型;
   - 编写复杂查询(连续聚合、降采样);
   - 与 DevOps 协作调优。

3. **AI 准备**:
   - 收集训练数据(质量预测、故障预测);
   - 评估 AI 模型效果(为 Phase 3 之后的 AI 项目铺路)。

**【任职要求】**

- 3-5 年数据分析经验;
- 熟练 SQL + Python;
- 熟悉制造业 KPI 体系;
- 熟悉 TDengine / InfluxDB / TimescaleDB(优先);
- **加分**:
  - 有 AI/ML 经验;
  - 懂统计学(基础 SPC 规则);
  - 懂 BI 工具(Metabase / Tableau / PowerBI)。

**【薪酬范围】** ¥15K-25K/月(13 薪)

## 附录:业务组的"反模式"

| 反模式 | 表现 | 提问方式 |
|---|---|---|
| 卖软件思维 | "我们这个功能很好,你必须用" | "如果客户不要这个功能,你怎么处理?" |
| 不下车间 | 只在办公室写文档 | "你之前在工厂蹲过吗?最长多久?" |
| 不会画流程 | 用文字写"业务流程" | "你能在白板上画一下采购流程吗?" |
| 听不懂业务术语 | "SPC 是什么?" | "你上次去的工厂,OEE 是怎么算的?" |
| 不会数据建模 | "这个让开发决定" | "工单和工序的关系是 1:N 还是 N:M?" |
| 怕客户 | 不敢跟高层开会 | "你之前跟总经理汇报过吗?什么感觉?" |
| 写不出文档 | "我脑子清楚就行" | "你写的最长需求文档多少页?" |
