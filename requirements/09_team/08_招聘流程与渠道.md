## 二、内部推荐机制

> **数据**:内部推荐的成功率是普通招聘的 **3 倍**,留任率高 50%。

### 2.1 推荐奖励

| 推荐岗位级别 | 推荐成功奖励 | 备注 |
|---|---|---|
| P4-P5(中级及以下) | ¥1,500/人 | 试用期通过后发 |
| P6(高级) | ¥3,000/人 | 试用期通过后发 |
| P7(资深/专家) | ¥5,000/人 | 试用期通过后发 |
| 关键岗位(技术负责人、Flowable、架构师) | ¥10,000/人 | CEO 特批 |

> 试用期通过后发放 100%,转正后补发 0%(全部已发)。**提前离职追回**。

### 2.2 推荐人义务

- 推荐前需对候选人做初步了解(避免推荐"明显不合适"的人);
- 协助 HR 安排面试;
- 候选人入职后担任 1 个月"buddy"(非正式导师)。

## 三、招聘节奏(12 个月)

| 月份 | 招聘重点 |
|---|---|
| **M0(7 月)** | 技术负责人、Flowable BPM 工程师、后端核心、前端核心、产品经理、UI 设计师、法务(外包) |
| **M1(8 月)** | 制造业务架构师、后端工程师(×1-2)、前端工程师(×1)、DevOps、实施工程师 |
| **M2(9 月)** | 前端工程师(×1)、后端工程师(×1)、全栈(可选)、QA(可选)、HR |
| **M3-M6** | DBA、安全工程师、生产域 BA、质量域 BA、客户成功经理 |
| **M6-M9** | 设备域 BA、全栈工程师、UI 设计师(补)、行政 |
| **M9-M12** | 数据分析师、第二个客户成功、市场/销售(可选) |

## 四、面试流程(标准 4 轮)

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  HR 沟通(30 min)  →  技术/专业面试(60-90 min)         │
│         ↓                      ↓                       │
│    简历匹配 + 文化       专业知识 + 项目经验            │
│         ↓                      ↓                       │
│  跨部门面试(60 min)   →  CEO 终面(30 min)             │
│         ↓                      ↓                       │
│    协作能力 + 价值观      战略对齐 + 入职意愿           │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 4.1 各轮重点

| 轮次 | 重点 | 决策维度 | 决策人 |
|---|---|---|---|
| **HR 沟通** | 简历匹配、动机、薪资、文化 | 是否进入下一轮 | HR |
| **专业面试** | 专业知识、项目经验、问题解决 | 是否专业过关 | 部门负责人 / 资深工程师 |
| **跨部门面试** | 协作能力、沟通能力、价值观 | 是否文化匹配 | 协作部门负责人 |
| **CEO 终面** | 战略对齐、长期意愿、综合素质 | 是否发 offer | CEO |

### 4.2 关键岗位的特殊流程

| 关键岗位 | 特殊流程 |
|---|---|
| **技术负责人** | 加 1 轮"架构评审"(60-90 min,与现有架构师/CEO 共同评审) |
| **Flowable BPM 工程师** | 加 1 轮"BPMN 实战"(60 min,现场画流程 + 讲实现) |
| **制造业务架构师** | 加 1 轮"业务案例"(60 min,讲一个他做过的制造业 IT 项目) |
| **前端/后端核心** | 加 1 轮"代码 review"(60 min,提前发一份代码,现场讲) |
| **DevOps** | 加 1 轮"故障排查"(60 min,现场给一个故障场景,看排查思路) |
| **法务** | 加 1 轮"合同案例"(60 min,带 2 份他审过的合同,讲思路) |

### 5.1 评分标准

| 评分 | 含义 |
|---|---|
| **1(不推荐)** | 完全不匹配,基础知识/经验不达标 |
| **2(待定)** | 部分匹配,有明显短板 |
| **3(够用)** | 符合基本要求,能干活 |
| **4(优秀)** | 超出预期,有亮点 |
| **5(卓越)** | 远超预期,必招 |

> **决策原则**:总分 < 24 不通过;有任一维度 = 1 直接淘汰。

## 六、面试常见问题(按岗位)

### 6.1 产品经理

1. **行业理解**:
   - 你对制造业数字化转型的理解?MES/ERP/QMS 的区别?
   - 用友 U9 / 金蝶云·星瀚 / 鼎捷 ERP / 宝信 MES 你调研过哪些?优劣?

2. **需求分析**:
   - 如果你是我们的产品经理,第一版会做哪些功能?
   - 总经理和班组长需求冲突,你怎么处理?

3. **场景题**:
   - 如果客户投诉"工单流转太慢",你怎么分析?
   - 如果工单状态机有 8 个状态,你怎么设计 PRD?

### 6.2 前端工程师

1. **基础知识**:
   - React 19 的新特性(use + Server Components)?
   - TypeScript strict 模式的关键点?
   - 性能优化方法(useMemo / useCallback / React.memo / 虚拟列表)?

2. **项目经验**:
   - 你做过的最有挑战性的前端项目?
   - 复杂表单(react-hook-form + zod)怎么设计?

3. **场景题**:
   - 大屏(echarts)的性能优化?
   - 移动端 H5 的 1px border 怎么做?

### 6.3 后端工程师

1. **基础知识**:
   - NestJS 的 DI / Module / Lifecycle?
   - PostgreSQL 事务 / 索引 / 锁?
   - Prisma 的 N+1 问题怎么解决?

2. **架构设计**:
   - 怎么设计"工单"模块?状态机怎么存?
   - 怎么设计"权限"模块?RBAC vs ABAC?

3. **场景题**:
   - 如果告警中心写入量很大(每秒 1000 条),你怎么设计?
   - 如果 Flowable 调用很慢,你怎么排查?

### 6.4 Flowable BPM 工程师

1. **基础知识**:
   - BPMN 2.0 的 gateway / event / sub-process?
   - Flowable 的 job executor 调优?
   - DMN 决策表怎么设计?

2. **场景题**:
   - 请画一个"采购审批"流程(含分支、会签、超时)。
   - 业务表和流程表怎么桥接?

3. **项目经验**:
   - 你做过的 BPM 项目?解决了什么复杂问题?

### 6.5 制造业务架构师

1. **行业理解**:
   - 离散制造 vs 流程制造 vs 单元制造的区别?
   - MES 和 ERP 的边界?QMS 和 MES 的边界?
   - ISO 9001 / IATF 16949 的核心条款?

2. **业务能力**:
   - 工单的状态机(派工 → 接单 → 报工 → 完工 → 检验 → 入库)?
   - 排产的瓶颈识别?
   - OEE 的计算公式?怎么提升?
   - SPC 的 8 条判异规则?

3. **场景题**:
   - 客户说"我们要上 MES",你怎么调研?
   - 如果客户已经有 ERP,你怎么定位 MES?

### 6.6 DevOps

1. **基础知识**:
   - Docker / Docker Compose / K8s 的区别?
   - CI/CD 流水线设计?
   - Prometheus + Grafana 监控告警?

2. **场景题**:
   - 容器启动失败,怎么排查?
   - 数据库主从延迟,怎么处理?
   - 7×12 on-call 怎么排班?

3. **项目经验**:
   - 你做过的最复杂的部署架构?
   - 私有化部署经验?

### 6.7 实施工程师

1. **项目经验**:
   - 你做过的最大实施项目?周期?客户?金额?
   - 客户最棘手的问题是什么?你怎么解决的?

2. **沟通能力**:
   - 客户方项目经理难缠,你怎么处理?
   - 客户业务部门不配合,你怎么推动?

3. **场景题**:
   - 现场实施时,客户临时要求加功能,你怎么处理?
   - 上线当天出 BUG,你怎么应急?

## 七、入职流程(第一周)

| 天 | 内容 | 责任人 |
|---|---|---|
| **D1** | 入职手续(合同、社保、公积金)、领取设备(电脑/账号)、认识团队 | HR |
| **D2** | 公司介绍 + 项目介绍(看 9 份核心文档:`00_overview/` + `01_architecture/`) | PM |
| **D3** | 业务流程培训(经营/管理/生产/支持/附加 5 域 + 15 个业务角色) | 制造业务架构师 |
| **D4** | 系统使用培训(4 个前端应用 + 后端 Swagger + Flowable Modeler + Keycloak) | 各部门负责人 |
| **D5** | 第一次参与团队会议(Sprint 计划会 / 周会),融入团队 | 部门负责人 |
| **D1-D5** | 安排"buddy"(内部推荐人或同岗位资深) | 部门负责人 |
| **D7-D30** | 试用期第一阶段:小任务 + 学习 | 部门负责人 + buddy |

### 7.1 试用期管理

| 阶段 | 时长 | 关键动作 |
|---|---|---|
| **试用期** | 3 个月(管理层 / 关键岗位 6 个月) | 明确目标 + 导师带教 + 月度复盘 |
| **第 1 个月** | 月度 review | 适应团队 + 小任务完成度 |
| **第 2 个月** | 月度 review | 任务完成度 + 协作评价 |
| **第 3 个月末** | 转正答辩 | 提交转正答辩 PPT,部门负责人评议 |
| **第 6 个月末** | 关键岗位转正答辩 | 同上 |

## 九、招聘 SOP(给 HR)

### 9.1 接到需求

1. 部门负责人提"招聘需求单"(含岗位、级别、紧急度);
2. CEO 审批(关键岗位必须 CEO 审批);
3. HR 启动招聘。

### 9.2 简历筛选

1. 看经验(年限、行业、关键技能);
2. 看稳定性(每段工作时间 > 1 年,跳 3 跳以内);
3. 看项目经历(具体、有数据、不是套话);
4. 1 个工作日内给反馈(快!)。

### 9.3 约面试

1. HR 沟通(30 min,先电话/视频);
2. 通过 → 约专业面试(60-90 min,现场/视频);
3. 通过 → 约跨部门面试(60 min);
4. 通过 → 约 CEO 终面(30 min);
5. 每一轮 2 个工作日内给反馈。

### 9.4 发 offer

1. CEO 终面通过 → HR 谈薪(参考 [07 薪酬体系](./07_薪酬体系.md));
2. 候选人确认 → 发 offer letter(电子签);
3. 候选人签 offer → 约定入职日期;
4. 入职前 1 周发"入职准备邮件"(设备、账号、文档)。

## 十、招聘漏斗看板(飞书/钉钉)

> 维护一个看板,实时跟踪招聘进度。

```
需求池 → HR 沟通 → 专业面试 → 跨部门 → CEO 终面 → offer → 入职
[ 5 ]      [ 12 ]     [ 8 ]       [ 5 ]     [ 3 ]     [ 2 ]  [ 1 ]
```

| 阶段 | 转化率(健康) |
|---|---|
| 需求池 → HR 沟通 | 1.5:1(每个岗位 1.5 个候选人进入 HR 沟通) |
| HR 沟通 → 专业面试 | 1.2:1 |
| 专业面试 → 跨部门 | 1.5:1 |
| 跨部门 → CEO 终面 | 1.2:1 |
| CEO 终面 → offer | 1.2:1 |
| offer → 入职 | 1.3:1(70% 接受率) |

> **如果某一阶段转化率显著低于健康值**,要 review 原因(JD 不清晰?薪酬不匹配?面试官太严?)。
