# 09 绩效晋升与文化

> 本文档定义 GJMZZ-DP 项目的**绩效考核、晋升通道、股权激励、团队文化**。
> 配套:[01 总览](./01_团队组建总览.md) · [07 薪酬体系](./07_薪酬体系.md) · [08 招聘流程](./08_招聘流程与渠道.md)

## 一、绩效考核

### 1.1 考核周期

| 层级 | 周期 | 说明 |
|---|---|---|
| 高管(CEO / 联创) | 半年度 + 年度 | 看公司业绩 + 战略目标 |
| 部门负责人(技术负责人、架构师、PM 负责人) | 季度 + 年度 | 看部门业绩 + 团队建设 |
| 中层(P5-P6) | 月度 + 季度 + 年度 | 看任务完成度 + 协作 |
| 基层(P4) | 月度 + 季度 | 看任务完成度 |

### 1.2 考核维度

| 维度 | 权重 | 说明 |
|---|---|---|
| **业绩成果(OKR/KPI)** | 50% | 关键产出 + 目标完成度 |
| **专业能力** | 20% | 技能提升、解决问题能力、技术贡献 |
| **协作贡献** | 15% | 跨部门协作、知识分享、CR 质量 |
| **文化价值观** | 15% | 用户视角、团队精神、创业心态 |

### 1.3 评级与调薪

| 评级 | 占比目标 | 调薪幅度 | 说明 |
|---|---|---|---|
| **S(卓越)** | 10% | 20%+ 或晋升 | 远超目标,贡献突出 |
| **A(优秀)** | 30% | 10-15% | 超额完成 |
| **B(达标)** | 50% | 5-10% | 完成目标 |
| **C(待改进)** | 8% | 0-5% | 部分完成 |
| **D(不合格)** | 2% | 0 或淘汰 | 未完成 |

> **强制分布**:每次评级必须 S + A ≤ 40%,C + D ≥ 5%。避免"老好人文化"。

### 1.4 关键岗位的特殊考核

| 关键岗位 | 特殊指标 |
|---|---|
| **技术负责人** | 架构评审通过率、团队 OKR 完成度、技术债清理 |
| **Flowable BPM 工程师** | BPMN 模板数量、流程时效改进、平台稳定性 |
| **制造业务架构师** | 客户验收通过率、业务架构文档质量、售前支持 |
| **DevOps** | 系统可用率(SLA)、故障恢复时间、监控覆盖率 |

## 二、晋升通道

### 2.1 晋升路径

```
         ┌─────────────┐
         │  管理通道 M   │  CEO → VP → 总监 → 经理 → 主管
         └──────┬──────┘
                │
P4(初级)  →  P5(中级)  →  P6(高级)  →  P7(资深/专家)
         ┌──────┴──────┐
         │  专业通道 P   │  P4 → P5 → P6 → P7 → P8(首席,可选)
         └─────────────┘
         ┌─────────────┐
         │  业务通道 B   │  业务专员 → 业务骨干 → 业务专家 → 业务总监
         └─────────────┘
         ┌─────────────┐
         │  实施通道 I   │  实施助理 → 实施工程师 → 高级实施 → 实施专家
         └─────────────┘
```

> **通道可切换**:P5+ 员工可申请转通道(技术转管理、专业转业务),但需要通过评审。

### 2.2 晋升规则

| 维度 | 规则 |
|---|---|
| **频次** | 每年 2 次(4 月、10 月) |
| **资格** | 在当前级别满 1.5-2 年;连续 2 次评级 A 或 1 次 S |
| **流程** | 自我申报 → 直接上级评议 → 跨部门评议 → 晋升答辩 → CEO 审批 |
| **答辩** | 15-20 分钟 PPT 答辩(技术 / 管理 / 业务) |

### 2.3 晋升答辩模板

```
候选人:______  当前级别:______  目标级别:______

1. 关键贡献(过去 1.5-2 年,3-5 个 STAR 故事)
2. 能力提升(技术/管理/业务)
3. 对下一级别的规划
4. 对公司/团队的建议

评议:通过 / 待改进 / 不通过
```

### 2.4 关键岗位的"破格晋升"

> 关键岗位(技术负责人、Flowable BPM 工程师、制造业务架构师)允许**破格晋升**:
> - 不到 1.5 年可申请(但必须有显著贡献);
> - 跳过答辩(CEO 直接任命);
> - 但必须全员公示 + 接受 6 个月考核期。

## 三、股权激励

### 3.1 股权池设置

- 建议预留 **10-15%** 期权池(创业期精简,后续融资后扩);
- 主要用于:技术负责人、Flowable BPM 工程师、制造业务架构师、P6+ 核心;
- 4 年 vesting:满 1 年后开始,每年 25%。

### 3.2 期权授予对象(参考表)

| 对象 | 授予比例 | 说明 |
|---|---|---|
| CEO | 已有(创始人) | — |
| 技术负责人 | 1-2% | 核心 |
| Flowable BPM 工程师 | 0.5-1% | 关键岗位 |
| 制造业务架构师 | 0.5-1% | 关键岗位 |
| 产品经理(核心) | 0.3-0.5% | 重要 |
| P6+ 核心研发 | 0.2-0.5% | 重要 |
| 其他 P5+ 员工 | 0.1-0.3% | 激励 |

### 3.3 期权发放条件

- **授予**:转正后授予;
- **vesting**:每年 25%(满 1 年后开始);
- **行权价**:按授予时的公司估值(种子轮 / 天使轮);
- **离职**:未 vesting 部分收回;已 vesting 部分可行权(若公司同意)。

### 3.4 期权协议

> 期权必须签正式协议(由法务起草,参考标准 SAFE / 期权协议模板)。

## 四、团队文化建设

### 4.1 文化价值观

| 价值观 | 含义 | 行为体现 |
|---|---|---|
| **客户视角** | 站在客户(总经理/制造副总/班组长)角度思考 | 每月至少 1 次客户访谈或客户现场 |
| **极致专注** | 把核心功能做到极致 | 80% 精力在 20% 关键功能 |
| **协作共赢** | 跨部门互相成就 | 周会上主动分享、帮助他人 |
| **持续成长** | 永远比昨天进步一点 | 每月读 1 本书 / 1 门课 |
| **敢于试错** | 快速尝试、快速迭代 | 失败不指责,总结经验 |
| **下沉一线** | 经常去客户工厂(尤其业务/实施) | 每季度至少 1 次客户现场 |

### 4.2 团队活动

| 频率 | 活动 |
|---|---|
| **每周** | 周五下午茶 |
| **每月** | 月度团建(聚餐/运动/桌游) |
| **每季度** | 季度复盘会 + 户外团建 |
| **每年** | 年终总结 + 年度旅游 |
| **节日** | 春节、端午、中秋等节日福利 |
| **客户现场** | 团队每季度至少 1 次去客户工厂(去车间、走流程) |

### 4.3 学习成长机制

| 机制 | 说明 |
|---|---|
| **读书会** | 每月 1 次,技术/产品/管理/业务主题 |
| **内部分享** | 每周 1 次(午餐分享会) |
| **外部培训** | 公司报销,每年 2 次(每人上限 ¥5K) |
| **会议外派** | 每年 1 次行业大会(每人上限 ¥10K) |
| **导师制** | 每位新员工配 1 位导师(3 个月) |
| **行业认证** | 报销关键岗位的认证费用(PMP / CQE / ISO 内审员等) |

### 4.4 沟通机制

| 机制 | 频次 | 说明 |
|---|---|---|
| **每日站会** | 每日 15 分钟 | 前端/后端/BPM/QA 各小组 |
| **Sprint 计划会** | 双周 | 2 周一个 Sprint |
| **Sprint 评审会** | 双周 | 演示 + 反馈 |
| **Sprint 回顾会** | 双周 | 改进 |
| **月度全员会** | 每月 | CEO 讲公司进展 + Q&A |
| **季度复盘会** | 每季度 | OKR 复盘 + 战略调整 |
| **年度总结会** | 每年 12 月 | 全年复盘 + 下一年规划 |

## 五、留人策略

> 制造业 IT 的人才流失率约 15-20%(参考 2025 行业数据),要低于这个数。

### 5.1 留人四要素

| 要素 | 策略 |
|---|---|
| **薪酬有竞争力** | 市场中位数 ± 10%(参考 [07 薪酬体系](./07_薪酬体系.md)) |
| **成长看得见** | 明确的晋升通道 + 年度调薪 + 期权 |
| **工作有意义** | "我们在帮中国制造业数字化转型"(使命感) |
| **氛围舒服** | 扁平管理、尊重专业、不加班文化(项目忙时可调休) |

### 5.2 关键岗位的"留人红线"

| 关键岗位 | 关注信号 | 应对 |
|---|---|---|
| **技术负责人** | 抱怨决策被干预、招不到人 | 充分授权 + 协助招聘 |
| **Flowable BPM 工程师** | 长期单兵作战 | 增加 1 个副手(后端工程师可转) |
| **制造业务架构师** | 长期出差疲惫 | 控制出差频次 + 远程办公 |
| **DevOps** | 7×12 on-call 压力大 | 排班合理 + 故障奖励 |
| **实施工程师** | 客户刁难 | 客户成功 + 售前支持(避免单打独斗) |

### 5.3 离职面谈

> 离职前必做,收集真实反馈。

| 问题 | 目的 |
|---|---|
| 离开的真实原因是什么? | 改进留人策略 |
| 工作中最有成就感的是什么? | 保留好的做法 |
| 工作中最不满意的是什么? | 改进 |
| 会推荐朋友来吗? | 雇主品牌 NPS |
| 离职后去向? | 竞业 + 关系维护 |

## 六、试用期转正与淘汰

### 6.1 转正条件

- 完成试用期目标(部门负责人定);
- 月度 review 全部 ≥ B;
- 部门负责人 + HR 评议通过。

### 6.2 淘汰机制

| 阶段 | 触发条件 | 处理 |
|---|---|---|
| **试用期 1 个月** | 明显不适配 | HR 协商离职(赔偿 1 个月) |
| **试用期 3 个月** | 转正答辩不通过 | 延长试用期 1-3 个月 / 协商离职 |
| **转正后** | 连续 2 次 C 或 1 次 D | 绩效改进计划(PIP)1-3 个月 |
| **PIP 后** | 仍不达标 | 协商离职(赔偿 N+1) |

> **重要**:制造业 IT 圈子小,**口碑很重要**——离职要体面,该赔的赔,该给的给,不要省小钱。

## 七、附录:文化建设"反模式"

| 反模式 | 表现 | 应对 |
|---|---|---|
| 加班文化 | 晚上 10 点走是"不努力" | 明确"不强制加班",项目忙时可调休 |
| 老好人文化 | 评级全是 B,没人 S/A | 强制分布(S+A ≤ 40%) |
| 老板依赖 | "老板说什么就是什么" | 鼓励"敢于提不同意见" |
| 不会表扬 | 做了好事没人说 | 月度全员会公开表扬 |
| 推卸责任 | 出了事都怪别人 | 故障复盘(postmortem)对事不对人 |
| 不会批评 | 出了事不敢说 | 1v1 沟通 + 文档记录 |
| 朋友圈文化 | 周末团建强制参加 | 团建自愿,不强求 |
