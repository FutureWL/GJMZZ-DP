## 2. 为什么单独建一个目录

- `requirements/00_overview/03_三层组织与角色.md` 定义了**业务角色**(15 个),是平台的"用户";
- `requirements/08_role-journeys/` 是**角色端到端验收**;
- `09_team/` 则是**建设这些能力的人**(开发/产品/实施/运维团队),与业务角色一一对应但**职能不同**。

## 3. 一句话总结

> 12 个月内,组建一支 **14-22 人** 的"工业软件精兵":
> - **3-5 个核心技术骨干**(NestJS 全栈 / Flowable BPM / 工业集成 / DevOps / 业务架构师)
> - **4-6 个前端 + 后端**(React 19 / NestJS 10)
> - **2-3 个制造业领域专家**(生产/质量/设备经理,有工厂 IT 落地经验)
> - **2-3 个产品 + UI**(懂 B2B 复杂信息系统)
> - **2-3 个测试 + DevOps + 安全**(保障生产稳定)
> - **1-2 个实施 + 客户成功**(上门部署、培训、运维支持)

## 4. 与项目现状的关系

| 项目状态 | 团队能力需求 |
|---|---|
| 已有 5 个 App 的前端(portal-ui / mobile-portal-ui / cockpit-ui / mobile-app / factory-api)的 MVP | 需要 **补全 + 深化** 团队 |
| Keycloak SSO 已配 15 个种子用户 + 36 条菜单 + 角色映射 | 需要 **生产域 + 移动端 + 实施** 人员 |
| Flowable 已接 4 个 BPMN(simple_approval / qc_exception / multi_step / 流程桥) | 需要 **BPM 平台工程师** 深化 |
| Prisma 已落 `incident` 表,其余走 mock | 需要 **后端 + DBA** 把核心域(工单/质量/设备/告警)落库 |
| TDengine / MinIO / RabbitMQ 已起 | 需要 **DevOps + 时序数据工程师** 用起来 |
| 真实工厂 OT/SCADA 未对接 | **P1 关键缺口**:需要工业集成/OT 工程师 |
