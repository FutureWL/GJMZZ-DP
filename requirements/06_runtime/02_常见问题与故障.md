# 02 常见问题与故障

> 收录开发期/运行期常见错误及处理方式
> 单一案例的详细取证记录,放在 `assets/incidents/`(后续补)

## 1. 启动相关

### 1.1 端口冲突
- 现象:`bind: address already in use`
- 处理:停占用端口的进程,或修改根 `docker-compose.yml` 中对应映射
- 兜底:`docker compose down --remove-orphans && docker compose up -d --build`

### 1.2 Keycloak 启动慢 / realm 没导入
- 现象:Keycloak UI 可访问但没有 `factory-platform` realm
- 原因:导入任务未执行或 `infra/compose/sso-dev/realm` 路径错
- 处理:查 `docker compose logs keycloak | grep -i realm`;确认 `infra/compose/sso-dev/realm/factory-platform-realm.json` 存在

### 1.3 Postgres healthcheck 一直不通过
- 处理:查 `docker compose logs db`;`/var/lib/postgresql/data` 权限问题→`sudo chown -R 999:999 ./data/postgres`

### 1.4 factory-api 启动报 Prisma 找不到
- 处理:`pnpm -C apps/factory-api build` 重新生成 client(已含 `prisma generate`)

## 2. SSO / Keycloak

### 2.1 浏览器访问 sso.corp.aygjm.lan 502
- 原因:系统/软件代理接管
- 处理:把 `sso.corp.aygjm.lan` 加入代理"忽略/直连"列表

### 2.2 portal-ui 一直跳回登录页
- 原因:issuer 与浏览器地址不一致
- 处理:确认 `KC_HOSTNAME=sso.corp.aygjm.lan`;`AUTH_ISSUER` 与 `issuer` claim 一致;hosts 已加

### 2.3 silent-check-sso 404
- 原因:`public/silent-check-sso.html` 缺失
- 处理:把仓库已有的拷贝到对应 app 的 `public/`

## 3. 移动端

### 3.1 APK 启动白屏 ⚠️
- 现象:debug APK 启动后纯白屏
- 详细取证:仓库根 `debug-apk-white-screen.md`
- 假设(可证伪):
  1. `assets/public/index.html` 缺失(构建后未复制)
  2. JS 报错 → logcat 过滤 `Chromium|JS error|应用包名`
  3. 路由 basename 不匹配(Web `/m/app/` vs Android `https://localhost/`)
  4. Keycloak 静默回调阻塞
  5. Capacitor/Activity 初始化失败(FATAL EXCEPTION)
- 取证步骤:
  ```bash
  adb logcat -v color | grep -iE "capacitor|chromium|webview|js|应用包名"
  # Chrome 远端调试
  chrome://inspect/#devices
  ```

### 3.2 Capacitor sync 后 web 资源没更新
- 处理:删除 `android/app/src/main/assets/public` 后再 `pnpm android:sync`

## 4. 后端

### 4.1 factory-api 报 "no pg_hba.conf entry"
- 原因:不是连 `db` 而连了 `localhost`
- 处理:在容器内 `DATABASE_URL=postgres://factory:factory@db:5432/factory`

### 4.2 Swagger 加载失败
- 处理:确认 `factory-api` 启动正常;`http://localhost:33704/docs`

### 4.3 Flowable 端口与 gateway https 冲突
- 现象:`33721` 既被 flowable-idm 占用又被 gateway https 占用
- 处理:调试期直接访问 `localhost:33721`(flowable-idm);gateway https 改 `33722`?或按需调整

## 5. 前端构建

### 5.1 pnpm install 报 "not found: @factory/mock-data"
- 处理:确认根 `pnpm-workspace.yaml` 注册了 `packages/*`;`pnpm install` 在根目录执行

### 5.2 Vite 报 "Cannot find module @factory/mock-data"
- 处理:删除 `node_modules/.vite` 后重启 dev;确认 `packages/mock-data/package.json` 的 `exports."."` 指向 `src/index.ts`

## 6. 调试技巧

- **日志聚合**:`docker compose logs -f --tail=200`(根 `package.json` 已包装为 `pnpm docker:logs`)
- **进入 Postgres**:`docker compose exec db psql -U factory -d factory`
- **进入 Keycloak**:`docker compose exec keycloak /bin/bash`
- **重置某个服务数据**:删除对应 `./data/<service>` 后 `up -d`
