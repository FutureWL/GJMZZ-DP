# 01 本地开发流程

> 适用:Windows / macOS / Linux 本地开发
> 假设:已安装 Docker Desktop(或 Docker Engine) + pnpm 11.x + Node 22

## 1. 一次性准备

```bash
# 1. 进入仓库
cd /root/DataDisk/workspace/GJMZZ-DP

# 2. 复制 .env(已存在可跳过)
cp .env.example .env

# 3. (Windows 推荐) 添加 hosts
# 127.0.0.1 sso.corp.aygjm.lan
```

## 2. 一键启动(全栈)

```bash
docker compose up -d --build
docker compose ps
```

启动后访问:
- `http://localhost:33700/`(统一入口)
- `http://sso.corp.aygjm.lan:18080/`(SSO)
- `http://localhost:33706/`(CloudBeaver)
- `http://localhost:33704/docs`(API 文档)

## 3. 单服务开发(热重载)

```bash
# 启动依赖(数据库、SSO、中间件)
docker compose up -d db db-init-keycloak db-init-flowable keycloak redis rabbitmq minio tdengine cloudbeaver

# PC 门户(Vite,5173)
pnpm -C apps/portal-ui dev

# 移动门户(Vite,5180)
pnpm -C apps/mobile-portal-ui dev

# 驾驶舱(Vite,默认 5173,如冲突改端口)
pnpm -C apps/cockpit-ui dev

# 移动 APP(Vite,5181)
pnpm -C apps/mobile-app dev

# 后端(NestJS watch,3001)
pnpm -C apps/factory-api dev
```

> 前端通过网关 33700 调用 API(线上行为);本地直连 `localhost:33704` 也可。
> 注意:前端代码中,Keycloak issuer 是 `sso.corp.aygjm.lan`,**直连要确保 hosts 已加**。

## 4. 调试移动 APP(Android)

```bash
# 1. 启动 web 资源
pnpm -C apps/mobile-app dev

# 2. 让 Android WebView 指向 Vite 开发服务器
# 修改 capacitor.config.ts: server.url = 'http://10.0.2.2:5181'  (模拟器访问宿主机)
# 模拟器: 10.0.2.2 = 宿主机
# 真机:   用电脑 IP

# 3. 同步 + 打开
pnpm -C apps/mobile-app android:sync
pnpm -C apps/mobile-app android:open
```

详见 `apps/mobile-app/DEPLOYMENT.md` 与 `TROUBLESHOOTING.md`。

## 5. 数据库初始化

```bash
# factory-api 启动时自动 prisma generate;首次 build 后,使用 db push 同步表结构
docker compose exec factory-api pnpm prisma db push
```

## 6. 常用 docker 命令

```bash
# 查所有容器状态
docker compose ps

# 看某个服务日志
docker compose logs -f factory-api

# 进容器
docker compose exec factory-api sh

# 拉取新依赖后重新构建
docker compose up -d --build portal-ui

# 停止并清理(警告:会清容器)
docker compose down

# 停止并清理卷(警告:会清所有数据)
docker compose down -v
```

## 7. pnpm workspace 常用

```bash
# 给所有 app 装依赖
pnpm install

# 给指定 app 加依赖(写入对应 package.json)
pnpm -C apps/portal-ui add react-hook-form

# 公共包(@factory/mock-data)被修改后,无需重新安装,前端 dev 会自动监听 TS 源
```
