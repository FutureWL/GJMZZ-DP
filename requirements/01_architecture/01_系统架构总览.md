## 3. 前端 → 后端的请求路径

前端静态资源 → `gateway(/33700)` 反代到对应容器;API 调用约定走 `factory-api` 的 `/api/*` 路径(具体路径见 `apps/portal-ui/src/app/api/` 与 Swagger `http://localhost:33704/docs`)。

> 本期前端默认仍多走 `packages/mock-data` 的样例数据(没有走真实后端),后端 NestJS 的 controllers 当前覆盖:`auth`、`menus`、`factories`、`incidents`、`workflow`(Flowable 代理)、`profiles`、`health`、`db`、`prisma`、`app`。

## 4. SSO 流(单点登录)

```
Browser  → portal-ui(未登录) → 重定向到 Keycloak 登录页
Keycloak → 登录成功 → 回调到 portal-ui(带 code)
portal-ui → 用 keycloak-js 换取 access_token → 注入请求
factory-api → 校验 Bearer token(jose) → 放行 / 拒绝
```

详细设计见 `04_SSO与权限设计.md`。

## 5. 关键技术决策(KT Decision)

| 议题 | 决策 | 原因 |
|---|---|---|
| 是否引入微服务 | 否,单 NestJS monolith | MVP 阶段节省运维 |
| 是否多 Postgres 实例 | 否,单实例多 database | 本机资源 + 数据目录冲突规避 |
| 前端框架 | React + Vite | 与设计稿/组件库生态成熟 |
| 移动端 | H5 + 原生双形态 | H5 走 `mobile-portal-ui`,原生用 `mobile-app`(Capacitor) |
| 状态管理 | zustand(移动) / 组件 state(PC) | PC 端域间不重状态,业务相对独立 |
| 流程引擎 | Flowable 7(官方镜像) | 4 个 App 一起跑,直接使用 |
| ORM | Prisma | 类型安全,迁移友好 |
| 图表 | echarts(portal-ui) / recharts(cockpit-ui) | echarts 适合复杂大屏,recharts 适合轻量看板 |
| 网关 | Nginx | 已有镜像,统一 80/443 |

## 6. 不在架构内的内容(避免误用)

- 没有 Service Mesh(Istio/Linkerd)
- 没有 API Gateway 业务级(Kong/APISIX)
- 没有分布式追踪(Jaeger/SkyWalking)
- 没有 K8s manifest(只用 Docker Compose)
- 没有配置中心(Nacos/Apollo),统一 `.env`

如未来需要扩展,见 `07_changelog/CHANGELOG.md` 的"待评审"区。
