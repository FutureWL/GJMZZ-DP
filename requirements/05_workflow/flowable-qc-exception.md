# 质量异常处理(QC Exception)流程

> 引用自:`docs/workflow/flowable-qc-exception.md`(原仓库根 docs/workflow)
> 用途:出现质量异常时的处置流程(发现 → 隔离 → 评审 → 处置 → 关闭)

## 1. 背景

- 触发:质量检验任务结果为 `ng` / `hold`、或现场反馈不良
- 发起人:检验员、班组长、客户端
- SLA:8 小时内首次响应(占位)

## 2. 节点(占位,待补 BPMN)

```
发现(检验员/班组长)
   └─ 隔离(质量工程师,标记 hold/可疑批次)
        └─ 评审(质量经理 + 生产经理,确定根因)
             ├─ 返工(回到工单)
             ├─ 报废(库存扣减)
             └─ 让步接收(特批,需总经理)
                  └─ 关闭(留痕,数据回流 SPC)
```

## 3. 表单字段

- 不良数 / 批次号 / 工单 id
- 不良类型(外观/尺寸/性能/其他)
- 临时措施
- 根本原因(评审后)
- 处置(返工/报废/让步)
- 附件(照片/报告)

## 4. 边界

- **不接 Flowable**:简单异常走维修/事故的内置状态机
- **接 Flowable**:跨域(生产+质量+采购)协作 + 需要催办/抄送
