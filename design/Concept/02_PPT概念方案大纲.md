# 02 PPT 概念方案大纲（建议）

## 0. 封面
- 项目名：高精密制造信息化平台（视觉方案）
- 版本号/日期/负责人

## 1. 背景与目标
- 为什么要做：统一三端、提升现场可视与处置效率、支持数据驱动
- 设计范围：仅视觉与界面规范（不含后端/数据建模/功能实现）

## 2. 设计原则
- 可靠优先：告警优先、状态冗余表达、减少误操作
- 清晰优先：层级、对齐、密度控制、可读性
- 复用优先：模板化页面结构与组件库
- 三端一致：语义、命名、状态体系一致

## 3. 三端信息架构（概览）
- PC：主操作入口（列表/详情/处置）
- 大屏：监控与指挥（实时、聚合、告警突出）
- 移动：处置闭环（告警/任务/扫码）

## 3.1 组织格局矩阵（五域×三层）
- 纵向五域：经营 / 管理 / 生产 / 支持 / 附加
- 横向三层：高层 / 中层 / 基层（信息偏好从“摘要指标”到“闭环执行”）
- 建议展示一张二维矩阵：标注各域的系统边界与典型入口

## 3.2 五域门户（边界与联动）
- 经营门户：经营驾驶舱与分析（以结果与趋势为主，占位口径）
- 管理门户：计划运营 + 流程合规 + 供应链管理（采购/SRM/外协/审批）
- 生产门户：生产/质量/设备/告警/追溯（强现场处置）
- 支持门户：行政/人事/财务/审计/体系/安保/IT/数据安全/EHS（以服务台与SLA闭环为主）
- 附加门户：人才发展中心/党群/工会/妇联等专项机构（触达与活动为主）
- 联动原则：共享组织范围/时间范围/审批中心/消息待办；全局搜索按域分组并可跨域跳转

## 4. 视觉方向与主题
- 企业中性风定义（关键词与样例）
- 浅色主题（办公）关键页面展示
- 深色主题（大屏/夜班）关键页面展示
- 状态语义体系（设备状态、告警等级、质量等级）

## 5. 设计系统摘要
- Token：色彩/字体/间距/圆角/阴影（含浅/深映射）
- 关键组件：导航、筛选、表格、告警、工单卡、追溯链路
- 图表规范：配色、阈值、单位、密度、可读性规则

## 6. 核心场景演示（按角色）
- 生产管理：总览 → 排程 → 工单详情 → 异常
- 质量工程：检验任务 → SPC → 追溯链路
- 设备/维修：设备监控 → OEE → 维修工单
- 车间值班：大屏告警墙 → 移动端处置闭环
- 高层：经营驾驶舱（摘要）→ 风险/异常 → 下钻到工厂/部门 → 审批中心
- 中层：采购闭环（PR→审批→RFQ→定标→PO）→ 供应商准入 → 外协交付概览
- 基层（支持域示例）：IT 工单（提交→受理→处理→验收→关闭）

- 页面模板复用策略（看板/列表/详情/配置）
- 后续对齐组件库（如需）的方法：命名映射、组件差异表
