# 01 组织格局矩阵与角色映射（五域×三层）

## 1. 目的
- 用统一的组织格局框架，解释平台为什么分成“五域门户”，以及每类人进入平台主要看什么、做什么
- 让视觉原型在“页面长相”之外，具备可落地的角色入口与导航逻辑（不做权限实现，只做模板）

## 2. 五域定义（纵向）
### 2.1 经营（Business）
- 核心：增长与结果（订单/收入/回款/毛利/客户）
- 典型对象：客户、订单、合同、回款、渠道
- 典型行为：看经营驾驶舱、看预测与趋势、做重大决策与审批

### 2.2 管理（Management）
- 核心：计划运营 + 流程合规
- 典型对象：目标/KPI、预算、经营计划、制度流程、内控审计、整改
- 典型行为：看达成率与差距、推动闭环、控制风险与超时

### 2.3 生产（Production）
- 核心：现场交付与质量（排程/执行/设备/告警/追溯）
- 典型对象：工单、批次/序列、设备、检验、告警
- 典型行为：处置异常、推进工单、保障设备与质量

### 2.4 支持（Support）
- 核心：共享服务与保障能力（行政/人事/财务/审计/体系/安保/信息IT/数据安全/EHS）
- 典型对象：服务请求/工单、台账、事件、合规项、培训与整改
- 典型行为：受理与闭环、SLA 与风险提醒、审计与留痕

### 2.5 附加（Additional）
- 核心：组织文化与专项机构（人才发展中心、党群、工会、妇联等）
- 典型对象：活动、培训、公告、组织信息、关怀项目
- 典型行为：通知触达、活动管理、培训与发展（与支持域“人事/培训”可做边界说明）

## 3. 三层定义（横向）
| 层级 | 信息偏好 | 典型页面形态 | 常见交互 |
|---|---|---|---|
| 高层 | 摘要、趋势、异常、风险 | 驾驶舱/大数字/KPI卡 | 下钻到域/部门/工厂，审批 |
| 中层 | 对标、结构、闭环、资源 | 部门看板+列表明细 | 分配任务、推动闭环、看超时 |
| 基层 | 任务、步骤、操作 | 列表/详情/表单/工单 | 处理、记录、提交、留痕 |

## 4. 角色样例（每层 5 个）
### 4.1 高层（5）
- 总经理
- 常务副总经理
- 营销副总（分管市场/销售）
- 制造副总（分管生产/质量/设备）
- 财务/风控负责人（分管财务/审计/合规/内控）

### 4.2 中层（5）
- 生产经理（或制造部经理）
- 质量经理
- 采购经理
- 设备/维护经理
- 信息化/IT 经理（含数据安全接口人）

### 4.3 基层（5）
- 班组长
- 计划员/调度员
- 检验员/质量工程助理
- 维修技师
- 物流/库管操作员

## 5. 角色→门户→默认落地页（模板）
说明：同一人可能属于多个域，默认落地页以“主要职责域”为主，其余域通过门户切换进入。

| 角色 | 主要域门户 | 默认落地页 | 常用路径（原型演示用） |
|---|---|---|---|
| 总经理 | 经营/管理 | 经营驾驶舱（集团/工厂切换） | 驾驶舱 → 异常摘要 → 下钻到部门/工厂 → 审批中心 |
| 营销副总 | 经营 | 经营驾驶舱（营销） | 驾驶舱 → 订单/回款趋势（占位） → 大额审批（占位） |
| 制造副总 | 生产/管理 | 生产总览（工厂级） | 生产总览 → 告警中心 → 下钻产线 → 处置闭环 |
| 财务/风控负责人 | 管理/支持 | 管理驾驶舱（风险/超时） | 风险/超时 → 审批中心 → 审计整改（占位） |
| 生产经理 | 生产 | 产线/车间看板 | 看板 → 工单列表 → 工单详情 → 异常/停线记录 |
| 质量经理 | 生产/管理 | 质量看板 | 质量看板 → SPC → 追溯 → 整改闭环（占位） |
| 采购经理 | 管理/经营 | 采购驾驶舱/待办 | PR 列表 → 发起审批 → RFQ 比价 → 定标 → PO |
| 设备经理 | 生产 | 设备监控/OEE | 设备监控 → 告警 → 维修工单 → 完工留痕 |
| IT 经理 | 支持 | 支持门户首页（服务台） | IT 工单 → 受理 → 处理 → 验收 → 关闭 |
| 班组长 | 生产 | 班组任务与告警 | 告警 → 确认 → 处置留痕 → 关闭 |
| 计划员 | 生产/管理 | 排程（甘特） | 排程 → 工单详情 → 交期风险提示（占位） |
| 检验员 | 生产 | 检验任务列表 | 任务列表 → 记录详情 → 判定 → 附件留痕 |
| 维修技师 | 生产/支持 | 维修任务（移动端/PC） | 任务 → 到场 → 完工 → 用料/工时留痕 |
| 物流/库管 | 生产/支持 | 物流台账（占位） | 入库/出库（占位）→ 关联工单/批次（占位） |

## 6. 跨域统一能力（视觉层）
- 组织范围：集团/工厂/部门/产线的统一选择器（顶部全局）
- 时间范围：统一时间控件（经营/管理偏月度，生产偏小时/日）
- 审批中心：统一待办/已办/抄送（跨域复用）
- 消息与提醒：告警（生产）与待办（管理/支持）并存，但样式体系一致
- 审计与留痕：关键动作统一留痕区域（备注/附件/时间线占位）
