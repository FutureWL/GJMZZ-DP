# 00 交互流转说明（PC / 大屏 / 移动）

## 1. 目标
- 让高保真原型具备“可演示”的最小交互闭环
- 统一三端关键动作的交互模式（筛选、查看详情、处置、追溯）

## 2. 全局交互（PC Web）
### 2.1 筛选与查询
- 页面通用筛选区结构：时间范围 + 组织范围（工厂/部门/产线）+ 关键字段（设备/工单/批次）+ 高级筛选
- 高级筛选：抽屉形式，支持保存为“常用筛选”
- 列表默认排序：告警按“严重度→时间”；工单按“状态→计划开始”

### 2.2 详情呈现
- 轻详情：抽屉（不离开列表、用于快速查看与处置）
- 重详情：详情页（包含多 Tab：概览/明细/关联对象/历史）

### 2.3 告警处置动作
- 统一动作：确认、转派、升级、关闭
- 处置留痕：动作后弹出填写面板（原因/措施/备注/附件位）
- 防误触：关闭告警必须二次确认（弹窗）

### 2.4 审批中心通用交互（管理/支持域）
- 审批入口：页面详情内“发起审批”按钮 + 全局“审批中心（待办）”
- 审批详情结构：流程节点（时间轴或流程图）+ 当前节点处理区 + 历史意见
- 常见动作：同意、驳回、退回、转交、加签（视觉位即可）
- 超时表达：`flow-overdue` 标签 + 倒计时/已超时文案位

## 3. 大屏交互（最小化）
- 信息展示优先，交互只保留：
  - 自动轮播开关（可选）
  - 模式切换（工厂/产线/告警墙）
  - 点击产线卡片进入产线屏（可选）
- 告警墙规则：
  - 严重告警固定置顶
  - 最新告警滚动（滚动速度可控）
  - 声光提示位（仅做视觉占位，不必实现）

## 4. 移动端交互（处置闭环）
### 4.1 告警闭环（P0）
1) 告警列表：默认只显示“未关闭”，支持严重度/产线/设备筛选
2) 告警详情：关键字段置顶（严重度、设备、开始时间、当前状态）
3) 处置动作：确认/转派/关闭
4) 留痕提交：原因、措施、备注、图片（可选）

### 4.2 工单闭环（P0）
1) 工单列表：待接单/进行中/待验收/已完成
2) 工单详情：任务说明、设备、位置、SLA/超时标签位
3) 状态推进：接单 → 到场 → 完工
4) 完工记录：耗材/工时/备注（视觉位）

### 4.3 扫码追溯（P0）
1) 扫码入口：从“任务/告警”页进入
2) 识别结果：批次/序列/工单/设备其中之一
3) 追溯详情：链路节点 + 关键检验结果摘要 + 关联工单/设备跳转

## 5. 三条演示路径（用于原型点击串联）
### Path A：告警处置
- PC：告警中心 → 告警详情抽屉 → 确认/转派 → 留痕 → 关闭
- Mobile：告警列表 → 告警详情 → 关闭 → 留痕

### Path B：维修工单
- PC：维修工单列表 → 工单详情 → 派工/状态推进（视觉位）
- Mobile：任务列表 → 工单详情 → 接单/到场/完工 → 完工记录

### Path C：追溯查询
- PC：全局搜索（批次/序列）→ 追溯结果 → 链路详情 → 关联对象跳转
- Mobile：扫码 → 追溯详情 → 关联对象

### Path D：采购闭环（决策门户）
- PC：PR 列表 → PR 详情 → 发起审批 → 审批中心（待办）→ RFQ/比价详情 → 定标确认 → PO 详情（收货节点位）

### Path E：供应商准入（决策门户）
- PC：供应商列表 → 准入申请 → 发起审批 → 审批中心 → 供应商详情（证照/风险/绩效）

### Path F：招聘编制（决策门户）
- PC：编制与岗位总览 → 招聘需求申请 → 发起审批 → 审批中心 → 岗位列表 → 候选人漏斗

### Path G：IT 工单闭环（支持门户）
- PC：IT 工单列表 → 工单详情 → 受理 → 处理 → 验收 → 关闭（占位）

## 6. 状态集合（用于原型统一标签）
- 告警：未确认 / 已确认 / 处理中 / 已关闭
- 工单：待接单 / 已接单 / 已到场 / 进行中 / 待验收 / 已完成
- 设备：运行 / 待机 / 停机 / 故障 / 离线 / 维护中 / 告警
- 审批：草稿 / 审批中 / 已通过 / 已驳回 / 已取消 / 超时
